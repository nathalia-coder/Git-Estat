3  Lista de Comandos Importantes

Atenção!!

Os códigos abaixo foram pensados para facilitar a sua vida.

Copie-os aqui e cole no terminal usando Ctrl + Shift + V

3.1 Comandos linux

É essencial ter uma base de comandos do linux para conseguir trabalhar no CMD de forma direta. Não só o prompt de comandos do dispositivo, mas também o git bash, o terminal do R e do Jupyter notebook e do software Anaconda, são exemplos de terminais que são usados com as linguagens do linux.

Por meio desses atalhos do CMD, conseguimos acompanhar melhor quais arquivos estão numa pasta, e assim, agilizar o versionamento de arquivos.

Abaixo, tem-se exemplos do uso de alguns códigos no terminal.

  • Para os terminais, conseguimos alterar um pouco a estética do prompt de comando usando o bash. Isso é comum no linux, e facilita a visualização. Mas não muda nada estar ou não com o bash ativado.
bash

Para sair do modo bash:

exit
  • Listagem dos arquivos na pasta
ls
ls -l
  • Para ver listagem dos arquivos na pasta que estão ocultos, normalmente a pasta .git criada pelo próprio sistema GIT para guardar os metadados de versionamento
ls -la
  • Printa caminho do diretório atual (print work directory):
pwd
  • Criar um diretório localmente (make directory):
mkdir nome-diretorio
  • Mudar de diretório (change directory)
cd nome-diretorio/
  • Voltar para o diretório anterior
cd ..

Comandos úteis do Vim:

  • Criar um arquivo simples direto do terminal

vim arquivo.extensao

Exemplo:

vim arquivo.py

3.2 Comandos git

  • Visualizar versão do git instalado no computador local:
git --version
  • Inicialização de repositório local para monitoramento
git init
  • Clonar um repositório remoto:
git clone https://github.com/gurkanakdeniz/example-flask-crud.git
  • Adicionar seu repositório remoto. Por padrão, nomeamos esse repositório como origin:
git remote add origing link_do_seu_repositorio
git origin

Visualizar repositórios remotos associados:

git remote -v
  • Criar nova branch:
git checkout -b novabranch
  • Entrar no ambiente de desenvolvimento de uma branch específica
git checkout nome_da_branch

Exemplo: git checkout main ou git checkout develop

  • Mostrar em qual branch estamos trabalhando:
git branch
  • Verificar qual a situação atual das alterações. Status de versionamento:
git status
  • Mostra diferença do que foi alterado em um arquivo versionado. Exibe em vermelho e começando com um - as linhas removidas e em verde começando com um + as novas linhas
git diff
  • Adicionar arquivo para ser monitorado

Podemos adcionar todo diretório para ser monitorado:

git add .

Ou apenas um arquivo (é importante colocar sua extensão)

git add meu-arquivo.txt
  • Excluir um arquivo (remove):
git rm
  • Desfazer marcação de um arquivo que foi monitorado (em add):
git restore
  • Commitar um arquivo com mensagem de descrição (boa prática):
git commit nome-arquivo -m "Descricao aqui"

Para comitar tudo, usar a flag -a que indica all:

git commit -a
  • Enviando um arquivo local para o remoto = PUSH
git push nome-da-pasta-remota nome-da-branch
Nota:

o Git geralmente pergunta se você quer definir origin (ou seja, origin main) como o padrão, para que você não precise ficar escrevendo sempre git push origin main, por exemplo.

  • Atualiza seu repositório local com o arquivo mais recente do repositório remoto. Remoto para local = PULL
git pull
  • Voltar ao estado do último commit feito (caso o presente commit queira ser revertido):

Usa-se o id do commit aqui.

git reset id-do-commit
  • Reverter alterações feitas em um commit, parecido com voltar a versão anterior:
git revert
  • Histórico de commits, inclusive autor do commit:
git log

Histórico apenas para um arquivo especifico

git log --follow nome-arquivo
  • Importantíssimo: para arquivos que não desejamos que que façam parte do repositório em si, sendo apenas disponíveis localmente para quem estiver executando o sistema. nota: Vamos ver um exemplo posteriormente.
git add .gitignore
  • Consultar histórico do que foi feito na linha de comando:
git blame nome-arquivo
